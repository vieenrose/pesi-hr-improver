1首頁與考勤異常儀表板

登入 hr.pesi.com.tw 後，擴充功能會在原本首頁上疊加一層待辦提示，讓你一眼看到需要補件的考勤異常。

  • 頂端 「有 N 筆考勤異常待處理」 提示列，右側 批次處理► 按鈕。
  • 下方 考勤異常通知 清單：員工代號、姓名、考勤日期、過濾條件。
  • 藍色連結（如「有缺勤時數無請假單時數」）可直接點進對應的補件表單。
  • 右下角可切換 背景通知（見第 8 節）。
上一年度／本年度特休餘額仍顯示於頂端跑馬燈，此處以 ●天 遮蔽。

2請假申請（含 Google 日曆）

從首頁 請假／申請 或工具列選單的 請假 進入。表單代號 SW0009。

  1. 假別：從重新排序後的下拉挑選（特休、事假、病假…）。
  2. 開始／結束日期：用瀏覽器原生日期選擇器，不必手動換算民國年。
  3. 時段快捷：一鍵套用 全天／上半天／下半天。
  4. 請假事由：常用事由會被記住，下次以標籤一鍵帶入。
  5. （選用）勾選 同時建立 Google 日曆事件。
  6. 按 送出申請；系統原本的確認畫面照常顯示。
Google 日曆：送出時會另開一個「建立活動」預填分頁，並帶入標題、起訖時間、事由 —— 你需在該分頁自行按「儲存」。不涉及任何 Google 授權或 API。

3公出／出差申請

從首頁 公出/出差／申請 或工具列選單的 公出出差 進入。表單代號 SW0212。操作與請假相同，多一個「類別」選擇。

  • 類別：選擇「公出」或「出差」。
  • 日期、時間、時段快捷、事由、Google 日曆選項皆與請假一致。
原生日期／時間選擇器時段快捷事由記憶Google 日曆

4忘打卡／補卡申請

從首頁 忘打卡／申請 或工具列選單的 忘記打卡 進入。表單代號 SW0013。

  • 打卡類型：上班／下班。
  • 日期／時間：原生選擇器。
  • 事由：輸入補卡原因。

6查詢今日刷卡

在工具列選單按 查詢今日刷卡，擴充功能會在背景查出今天的刷卡資料並整理顯示。

  • 上班 → 下班：當日第一筆與最後一筆，上班藍色、下班紅色。
  • 下方列出今天所有進出時間，依時間排序、以圓點分隔。
  • 若當日尚無刷卡，顯示「查無資料」。
多筆進出（例如中午外出）也會完整列出，方便核對是否漏刷。

7EIP 預約管理

在工具列選單按 預約管理，進入 EIP（eip.pesi.com.tw）預約模組。從左側資源樹選擇 會議室 / 公務車 / 來賓車位，再開啟預約表單。

擴充功能在會議室預約表單上額外提供：

  • 預約日期 旁的日期選擇器。
  • 起訖 時間 的時間選擇器（對應原本 AM/PM 時／分下拉）。
  • 事由 常用文字的記憶標籤。
不會自動送出：擴充功能只填欄位，一律由你確認後自行按 EIP 的 OK。

8背景通知

擴充功能可在背景定時檢查考勤異常，並以桌面通知提醒；首頁右下角的 背景通知：開／關 可切換。

  • 首次啟用時，瀏覽器會要求 通知 權限。
  • 點擊桌面通知會帶你回到考勤異常清單。
  • 使用 storage、notifications、alarms 權限。

9常見問題

Google 日曆整合需要授權我的 Google 帳號嗎？

不需要。它只是組出一個 Google 日曆「建立活動」的預填網址並另開分頁，沒有任何 OAuth 或 API。你必須在該分頁自行按「儲存」，活動才會真正建立。

擴充功能會不會把我的密碼存起來？

不會。登入輔助只帶入 帳號與公司代號，絕不處理密碼 —— 密碼一律交給瀏覽器自己的密碼管理員。

停用擴充功能後表單會怎樣？

所有表單原封不動地還原成原本樣子。擴充功能只在你的瀏覽器端疊加介面，不更動伺服器。

某張表單的優化不見了？

若廠商（日城資訊）改了該表單的欄位 ID，該表單的優化可能需要更新；其餘表單不受影響、仍可正常運作。
