## 1. 首頁與考勤異常儀表板

登入 `https://hr.pesi.com.tw` 後，擴充功能會在原本的首頁上疊加：

- 頂端 **「有 N 筆考勤異常待處理」** 提示列，右側有 **批次處理►** 按鈕。
- 下方 **考勤異常通知** 清單：列出每一筆待補的異常（員工代號、姓名、考勤日期、過濾條件）。
- 藍色連結（例如「有缺勤時數無請假單時數」）可直接點擊，帶你到對應的補件表單。

![考勤異常儀表板](images/01-dashboard.png)

> 上一年度／本年度特休餘額仍顯示於頂端跑馬燈；上圖中已以 `●天` 遮蔽。

## 2. 請假申請（含 Google 日曆）

從首頁點 **請假 / 申請**，或[工具列選單](#5-工具列快捷選單)的 **請假**。

![請假申請表單 SW0009](images/02-leave-form.png)

操作步驟：

1. **假別**：從重新排序後的下拉選單挑選（特休、事假、病假…）。
2. **開始／結束日期**：使用瀏覽器原生日期選擇器（不必再手動換算民國年）。
3. **時間快捷**：一鍵套用 **全天 (09:00–18:00)**、**上半天 (09:00–13:00)**、
   **下半天 (14:00–18:00)**。
4. **請假事由**：輸入原因；常用事由會被記住，下次以標籤形式一鍵帶入。
5.（選用）勾選 **「同時建立 Google 日曆事件」** — 送出時會另開一個 Google 日曆
   分頁並帶入標題、起訖時間與事由，**你需在該分頁自行按「儲存」**（不涉及任何
   Google 授權或 API，詳見 [FAQ](#9-常見問題)）。
6. 按 **送出申請**。系統原本的確認／結果畫面會正常顯示。

> 送出後畫面會先還原成原生表單，讓系統自己的確認視窗不被遮住。

## 3. 公出／出差申請

從首頁點 **公出 / 出差 / 申請**，或工具列選單的 **公出出差**。

![公出／出差申請表單 SW0212](images/03-trip-form.png)

- **類別**：選擇「公出」或「出差」。
- 日期、時間、時段快捷、事由與 Google 日曆選項，操作方式同[請假申請](#2-請假申請含-google-日曆)。

## 4. 忘打卡／補卡申請

從首頁點 **忘打卡 / 申請**，或工具列選單的 **忘記打卡**。

- **打卡類型**：上班／下班。
- **日期／時間**：原生選擇器。
- **事由**：輸入補卡原因。

## 5. 工具列快捷選單

| 按鈕 | 功能 |
|------|------|
| **出缺勤系統** | 開啟 HR 首頁（已開啟則切換到該分頁） |
| **預約管理** | 開啟 EIP 預約模組 |
| **請假** | 直接開啟請假申請表單 |
| **公出出差** | 直接開啟公出／出差申請表單 |
| **忘記打卡** | 直接開啟補卡申請表單 |
| **查詢今日刷卡** | 顯示今天的刷卡進出紀錄 |

## 6. 查詢今日刷卡

在工具列選單按 **查詢今日刷卡**，擴充功能會在背景查詢今日的刷卡資料並顯示：

- **上班 <第一筆> → 下班 <最後一筆>**（上班時間藍色、下班時間紅色）。
- 下方列出今天所有進出時間（依時間排序、以圓點分隔）。

若當日尚無任何刷卡紀錄，會顯示「查無資料」。

_（截圖待補：`images/06-swipe.png`）_

## 7. EIP 預約管理

在工具列選單按 **預約管理**，會進入 EIP（`https://eip.pesi.com.tw`）的預約模組。
你可從左側資源樹選擇 **會議室 / 公務車 / 來賓車位**，再開啟預約表單。

擴充功能在會議室預約表單上額外提供：

- **預約日期** 旁的日期選擇器。
- 起訖 **時間** 的時間選擇器（對應原本的 AM/PM 時／分下拉）。
- **事由** 常用文字的記憶標籤。

> ⚠️ 本擴充功能**不會**自動送出任何預約 — 一律由你自行確認後按 EIP 的 **OK**。

_（截圖待補：`images/07-eip-booking.png`）_

## 8. 背景通知

擴充功能可在背景定時檢查考勤異常，並以桌面通知提醒。首頁右下角的
**「背景通知：開／關」** 可切換。首次啟用時，瀏覽器會要求 **通知** 權限。

- 點擊通知會帶你回到考勤異常清單。
- 此功能使用 `storage`、`notifications`、`alarms` 權限。

## 9. 常見問題

**Q：Google 日曆整合需要授權我的 Google 帳號嗎？**
不需要。它只是組出一個 Google 日曆「建立活動」的預填網址並另開分頁，沒有任何
OAuth 或 API。你必須在該分頁自行按「儲存」，活動才會真正建立。

**Q：擴充功能會不會把我的密碼存起來？**
不會。登入輔助只會帶入 **帳號與公司代號**，**絕不處理密碼** —— 密碼一律交由瀏覽器
自己的密碼管理員。

**Q：停用擴充功能後表單會怎樣？**
所有表單會原封不動地還原成原本的樣子；擴充功能只在你的瀏覽器端疊加介面，
不更動伺服器。

**Q：某張表單的優化不見了？**
若廠商（日城資訊）改了該表單的欄位 ID，該表單的優化可能需要更新；其餘表單
不受影響、仍可正常運作。
